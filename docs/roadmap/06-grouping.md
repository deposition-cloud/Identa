# Phase 6 — Grouping

## Goal

Organise repositories into brand families and generate cohesive identity
systems rather than individual logos.

## Deliverables

- **Family creation UI** — drag/select repos into a BrandFamily
- **Shared visual language** — generate a colour palette, icon style, and
  typography recommendation that applies across the family
- **Per-repo variation** — each repo gets a unique icon within the shared style
  (e.g. same colour palette, different symbol)
- **Consistency scoring** — measure visual coherence between family members and
  flag outliers

## Acceptance Criteria

- [ ] User can create a BrandFamily and add 3+ repos
- [ ] Generated logos for family members share visible stylistic coherence
- [ ] Consistency score is displayed and updates as logos change
